Tutorial

1. General

This tutorial consists of a series of example pages. The templates are intentionally kept very simple. There is only a short description of the example and the rendered output. To understand what is going on do the following:


  1. investigate the tab 'try out' in the detail editor on the right and change its values.
  2. In the detail editor click on "..." in the toolbar to open the template editor. In the template editor checkout the template tab to see the handlebars template. Also in the form tab checkout the attribute definitions.
  3. open the context menu of the page in the tree and choose open as json. This will give yoou an understanding of the actual content data.

Demo CMS

There are some notable differences between this demo cms deployed on localstorage and the cms deployed on file based storages like github and local file system. In this demo pages have an automatically generated id, name and a parent. The path is generated from name and parent. On file-like systems the id is the path.

2. First Steps

The first steps are very basic. Simply follow the links below and have a look at the detail editor and its template. There is usually a single field that can be edited. Remember that the attributes id, name and parent always present in a page's template.

If you think you messed up the content, then simply press reset store in the top toolbar.
